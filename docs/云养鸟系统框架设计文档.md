# 云养鸟系统框架设计文档

## 1. 项目概述

### 1.1 系统背景

本项目是"云雀 Lark"物联网系统的功能扩展，旨在提升用户参与度，通过简易的"云养鸟"演示功能，为养殖场提供展示亮点，同时为用户提供有趣的互动体验。

### 1.2 设计目标

- **轻量化**：基于现有 ESP32 + FastAPI + Vue 架构增量开发，不新增复杂技术栈
- **可落地**：1个月可完成演示开发，贴合大学生技术能力
- **互动性**：用户可认领幼鸟、远程投喂、查看成长记录
- **简化设计**：无付费/积分系统，仅做身份绑定和基础互动

### 1.3 现有系统架构

## 2. 云养鸟系统架构设计

### 2.1 系统定位与用户区分

#### 2.1.1 两大系统定位对比

| 维度        | 鸟场管理系统                | 云养鸟系统               |
| --------- | --------------------- | ------------------- |
| **目标用户**  | 鸟场内部工作人员              | 普通客户群体              |
| **核心价值**  | 运营管理效率                | 用户体验与互动             |
| **使用场景**  | 日常办公环境（PC端）           | **移动端碎片化时间**        |
| **设备访问**  | PC浏览器                 | **手机浏览器**           |
| **功能侧重**  | 设备管理、用户管理、数据统计        | 幼鸟认领、远程投喂、成长追踪      |
| **交互频率**  | 高频、持续使用               | 低频、间歇性使用            |
| **权限复杂度** | 多级权限（root/admin/user） | 单一用户角色（cloud\_user） |
| **设计风格**  | 功能导向、效率优先             | **体验导向、趣味性强、移动端优先** |

**系统区分机制**：

| 系统     | 入口路径       | 登录页面           | 用户角色      | 设备访问   |
| ------ | ---------- | -------------- | --------- | ------ |
| 鸟场管理系统 | `/`        | `/login`       | root/user | 所有设备   |
| 云养鸟系统  | `/cloud/*` | `/cloud/login` | clouduser | 仅认领的幼鸟 |

**统一域名保障措施**：

1. **统一入口**：同一域名 `lark.xxx.com`，通过路径区分系统
2. **独立登录页**：不同登录页面 `/login` vs `/cloud/login`
3. **独立路由**：管理端根路径 `/`，云养端 `/cloud/*`
4. **共享认证服务**：共用后端用户认证体系，通过角色区分权限
5. **复用前端代码**：云养前端放在 `frontend/cloud/` 目录，与管理后台共享基础组件和工具函数

-- 雏鸟信息表
CREATE TABLE birds (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    name VARCHAR(50) NOT NULL COMMENT '雏鸟名称',

    id INTEGER PRIMARY KEY AUTOINCREMENT,

    temperature DECIMAL(4,1) COMMENT '环境温度(°C)',
    humidity DECIMAL(4,1) COMMENT '环境湿度(%)',

    notes TEXT COMMENT '备注',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (bird_id) REFERENCES birds(id) ON DELETE CASCADE
);

-- 云养认领表
CREATE TABLE cloud_adoption (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    user_id INTEGER NOT NULL,
    bird_id INTEGER NOT NULL,

    FOREIGN KEY (user_id) REFERENCES users(id),

);

-- 投喂记录表
CREATE TABLE cloud_feed_log (
    id INTEGER PRIMARY KEY AUTOINCREMENT,

    FOREIGN KEY (user_id) REFERENCES users(id),

**鸟场管理系统（现有管理后台）**：

- **前端**：Vue 3 + Element Plus + TypeScript
- **后端**：FastAPI + SQLAlchemy + Pydantic
- **数据库**：SQLite + Redis
- **设备**：ESP32 + Arduino

**云养鸟系统（新增移动端）**：

- **前端**：Vue 3 + Vant 4 + TypeScript（放在 `frontend/cloud/`）
- **后端**：复用鸟场管理系统 FastAPI（同一后端，不同路由）
- **数据库**：复用 SQLite + Redis
- **设备**：复用 ESP32

## 4. 数据流设计

### 4.1 认领流程

```
用户点击"认领"
       │
       ▼
前端 POST /cloud/adopt {bird_id}
       │
       ▼
服务端检查幼鸟是否可认领
       │
       ├─── 已认领 ──► 返回错误
       │
       └─── 可认领 ──► 创建认领记录
                           │
                           ▼
                      返回成功
```

### 4.2 投喂流程

```
用户点击"远程投喂"
       │
       ▼
前端 POST /cloud/feed {bird_id}
       │
       ▼
Redis 检查今日投喂计数
       │
       ├─── 已达上限(1次) ──► 返回错误
       │
       └─── 未达上限 ──►
              │
              ▼
         发送投喂指令至ESP32
              │
              ▼
         ESP32 触发喂食器
              │
              ▼
         记录投喂日志
              │
              ▼
         Redis 计数+1
              │
              ▼
         返回成功
```

### 4.3 成长记录查询流程

```
用户进入"我的云养"页面
       │
       ▼
前端 GET /cloud/growth/{bird_id}
       │
       ▼
服务端查询该幼鸟所有投喂记录
       │
       ▼
关联传感器历史数据
       │
       ▼
按日期生成成长记录列表
       │
       ▼
返回成长记录
```

## 6. 接口规格

### 6.1 认领接口

**POST** `/cloud/adopt`

Request:

```json
{
  "bird_id": 1
}
```

Response:

```json
{
  "code": 200,
  "msg": "认领成功",
  "data": {
    "id": 1,
    "user_id": 1,
    "bird_id": 1,
    "adopt_time": "2024-04-01 10:00:00",
    "bird_name": "小云",
    "bird_species": "品种A"
  }
}
```

### 6.2 投喂接口

**POST** `/cloud/feed`

Request:

```json
{
  "bird_id": 1
}
```

Response:

```json
{
  "code": 200,
  "msg": "投喂成功",
  "data": {
    "feed_time": "2024-04-01 14:30:00",
    "remaining_feeds_today": 0

}
```

## 7. 安全性考虑

### 7.1 权限控制

- 所有云养接口需要登录后访问（复用现有 `GetCurrentUser` 依赖）
- 用户只能查看和操作自己认领的幼鸟

### 7.2 投喂限制

- 每日每只幼鸟仅限投喂 1 次
- 使用 Redis 进行计数，带过期时间

### 7.3 输入验证

- 幼鸟 ID 必须存在
- 用户 ID 从认证信息获取，不从请求参数传入
