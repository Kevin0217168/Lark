## 项目概述与背景
（为什么做，解决了什么痛点）

云雀（Lark）通过物联网技术（自动喂水喂食，图像采集和分析），同时为鸟厂养殖户和个人用户提供智慧饲养服务。

- 对于养殖户（B端），通过智慧喂水食系统和云养鸟直播宣传，对于鸟厂人工批量饲养看管难，和在新互联网时代鸟厂知名度低，直销难的的问题提供了解决方案

- 对于个人用户（C端），通过销售智能鸟笼，搭建平台对接个人用户与专业养殖户，为个人养殖时遇到的健康问题没有线上渠道咨询，长期出门爱鸟无人看管提供了解决方案

## 作品简介和功能
作品是我们完全自主设计的一套包含设备端，服务端，客户端的智慧养殖完整系统方案。

### 设备端：
包含摄像头模块和环境参数采集模块，可以采集鸟笼的实时图像信息和鸟笼中实时温度，湿度，空气质量环境参数，通过wifi连接互联网实时传输。

### 服务端：
云服务器和内容分发网络，搭建了后端服务系统和数据库，存储用户信息和传感器数据，对设备传回的数据进行分析，为用户提供异常预警和建议。（图像分析未完成, 结合小挑）

### 客户端：
网站前端和小程序前端程序，提供用户账号信息和设备管理，实时展示设备端采集的信息以及服务器的分析汇总数据。（个人用户和专业养殖户交流平台未完成）


## 系统方案与核心技术
整个系统方案分为设备端，服务端和客户端。对应嵌入式技术，后端技术和前端技术方案
### 嵌入式技术方案：
#### 硬件设备选型：
均采用市面成熟的模块，性能稳定，方便前期快速开发

- 单片机MCU：ESP32-WROOM模块，拥有240MHz主频和双核心，4MB flash和4MB PSRAM，完全胜任图像采集和网络传输的任务，模块集成wifi天线和屏蔽罩，有良好的射频性能，并且符合电磁屏蔽标准
- 摄像头模块：OV3660，3MP镜头模组，最高300万像素，带有闪光灯和自动增益，硬件JPEG编码可节省算力并减少网络传输带宽使用
- 环境监测模组：
    - SHT40工业级温湿度传感器，I2C总线通信，体积小，精度高，功耗低，宽电压工作范围
    - MQ135空气质量传感器，监测空气中氨气含量，提供数字和模拟输出，灵敏度好，寿命长，驱动电路简单

#### 软件实现方案：
采用ESP-IDF官方库 5.5.4版本，原生FreeRTOS双核调度

Linux Vscode开发环境，CMake编译

依赖库：
- 扩展库
    - espressif/esp32-camera
    - espressif/esp_websocket_client


### 后端技术方案：
- 基础设施：
来自阿里云
  - 轻量应用云服务器 两核2G 40G系统盘
系统镜像Linux Debian，通过ssh和sftp直接连接进行运维
  - 边缘内容分发网络（ESA）
通过在业务服务器与客户端之间建立内容分发网络，在客户端就近处进行数据缓存，加速用户访问；提前清洗访问请求，减少业务服务器压力；通过直接将域名解析到ESA上，避免业务服务器公网ip直接暴露，降低被攻击的风险
  - 域名解析：
注册域名mintlab.top，已完成工信部和公安备案，云雀物联网管理系统搭建在lark.mintlab.top域名下，后端api搭建在lark.mintlab.top/api下, 属于同源访问, 提升安全性
    - 工信部备案号: [豫ICP备2026008104号](https://beian.miit.gov.cn/)
    - 公安网备号: [豫公网安备41010502007493号](https://beian.mps.gov.cn/#/query/webSearch?code=41010502007493)
- 软件架构: 
    - 数据库: SQLite + sqlalchemy 本地连接, 开发速度快, 安全性高
    - 服务: 
        - python FastAPI + Gunicorn 部署 配合systemd系统服务管理器
        - Redis 中间件 多线程共享内存
        - logging 日志管理, 自动按天轮转, 日志分级
        - swagger-ui 自动生成接口文档
